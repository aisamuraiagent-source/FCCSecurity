# Security Policy

## Mission

FCC Security work is defensive, auditable, reversible, and human-controlled.

## Rules

- Stay inside the authorized repository or mirror.
- Prefer static analysis before execution.
- Do not execute offensive, destructive, furtive, or out-of-scope actions.
- Do not print, store, or request secrets.
- Do not add network, dependencies, backend, telemetry, hooks, automation, CI/CD, commit, push, or deploy without explicit approval.
- Treat public wording as a security boundary: claims must be factual, dated, scoped, and evidence-backed.
- Keep GitHub read-only planned until an owner/repo or remote is supplied and approved.

## Severity Handling

- LOW: document and monitor.
- MEDIUM: fix or explicitly accept risk.
- HIGH: block release until fixed and validated.
- CRITICAL: suspend affected flow, require formal human review, fix, validate, and document.
